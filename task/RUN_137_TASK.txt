A1. v2 no-marker first write
- Start qbind-node with --p2p-trust-bundle and a valid v2 ratification sidecar on a fresh --data-dir.
- Expected:
  - release binary exits through the intended bounded startup/evidence path used by the harness, or starts far enough to prove trust-bundle startup acceptance before harness shutdown;
  - log contains [run-136] startup --p2p-trust-bundle v2 ratification path SELECTED;
  - Run 055 sequence persistence succeeds before v2 marker persistence;
  - log contains [run-136] v2 authority-marker persisted;
  - <data_dir>/pqc_authority_state.json exists;
  - marker record_version is v2;
  - latest_authority_domain_sequence equals the v2 sidecar sequence;
  - last_update_source is StartupLoad / startup-load, matching the serialized spelling used by the implementation;
  - <data_dir>/pqc_trust_bundle_sequence.json exists;
  - no v1 marker persist log fires for the v2 scenario;
  - no DummySig / DummyKem / DummyAead active path appears.

A2. v1-to-v2 migration
- Seed an accepted v1 startup marker or use the existing v1 startup path to create one.
- Restart with a valid higher-sequence v2 startup bundle/sidecar.
- Expected:
  - v2 path selected;
  - migration accepted;
  - Run 055 sequence commit precedes v2 marker persistence;
  - marker becomes v2 on disk;
  - previous v1 marker bytes differ from final v2 marker bytes;
  - last_update_source is StartupLoad;
  - no v1-after-v2 downgrade is introduced.

A3. idempotent same-digest v2 startup
- Seed or first accept a v2 startup marker.
- Restart with the same v2 bundle/sidecar and same authority-domain sequence/digest.
- Expected:
  - v2 path selected;
  - startup succeeds/accepts;
  - marker compare returns idempotent behavior;
  - marker file bytes are byte-for-byte unchanged before vs after;
  - marker mtime should not be treated as authoritative, but capture it if useful;
  - no sequence rollback/equivocation error;
  - no unnecessary marker rewrite.

A4. higher-sequence v2 startup upgrade, if feasible
- Seed v2 seq=N.
- Restart with valid v2 seq=N+1 and corresponding bundle/sidecar.
- Expected:
  - v2 path selected;
  - higher sequence accepted;
  - Run 055 sequence commit precedes marker persistence;
  - marker advances to N+1;
  - last_update_source is StartupLoad.
