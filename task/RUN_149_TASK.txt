Required positive scenario matrix:

A1. DevNet valid staged v2 candidate applies through Run 070
- V0 sends valid v2 candidate over live 0x05.
- V1 validates, stages, then applies under explicit DevNet policy.
- Expected:
  - live 0x05 validation path selected;
  - staging occurs after validation;
  - peer-driven apply policy permits DevNet;
  - Run 070 ordering is observed:
    validate → snapshot previous → swap → evict_sessions → commit_sequence;
  - v2 marker persists only after sequence commit;
  - `pqc_trust_bundle_sequence.json` advances;
  - `pqc_authority_state.json` becomes/updates V2;
  - session eviction path is invoked according to existing Run 070 semantics;
  - no SIGHUP/reload-apply/startup apply path is accidentally selected.

A2. DevNet idempotent/already-staged candidate behavior
- Same candidate received/applied or staged twice.
- Expected:
  - deterministic already-staged / already-applied / idempotent behavior;
  - no unbounded queue growth;
  - no duplicate unsafe sequence write;
  - no conflicting marker rewrite.

A3. DevNet higher-sequence v2 candidate applies
- Local marker seq=N.
- Candidate seq=N+1.
- Expected:
  - sequence advances;
  - v2 marker advances only after commit;
  - live trust state changes only through Run 070.

A4. DevNet v2-after-v1 migration applies
- Local v1 marker exists.
- Candidate is valid v2 migration candidate.
- Expected:
  - apply succeeds under explicit policy;
  - marker becomes V2 only after commit;
  - no downgrade path introduced.

A5. TestNet apply path, if feasible with available fixtures
- Same as A1 but TestNet environment with explicit policy.
- Expected:
  - TestNet is not enabled by default;
  - with explicit TestNet policy, apply uses same Run 070 path.
- If TestNet fixture generation is not available, document limitation and retain source/test Run 148 coverage.

- Apply policy disabled.
- Expected:
  - no apply;
  - no live trust mutation;
  - no sequence write;
  - no marker write;
  - no session eviction.

R3. unstaged candidate cannot apply

  - no mutation.

  - no apply;
  - no marker/sequence write.

R5. lower-sequence candidate cannot apply

R8. wrong-domain candidate cannot apply

- Expected:
  - validation rejects or apply recheck refuses;
  - no mutation.
