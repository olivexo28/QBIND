Current accepted state:

Run 291 is accepted as PASS / source-test production durable replay backend implementation.

Run 291 pivoted from modeled-layer expansion to Red production-backend closure by adding the first real production durable replay RocksDB backend:

`crates/qbind-node/src/pqc_governance_production_durable_replay_rocksdb.rs`

The backend is real RocksDB persistence, not another in-memory model. It persists typed replay records to an on-disk RocksDB database, recovers across reopen, enforces schema/domain binding, uses atomic `WriteBatch` writes, verifies record digests on read, enforces `Observed → Consumed` stage ordering, supports idempotency, refuses equivocation, fails closed on corruption/wrong-domain/partial-write state, uses deterministic SHA3-256 canonical length-prefixed digests, and never silently falls back to in-memory.

Run 291 is source/test only. The backend is not wired into the production binary. No CLI flag was added. MainNet identities are refused at open. The default policy remains Disabled. Full C4 remains OPEN. C5 remains OPEN.

B. Rejection / fail-closed release checks

* default Disabled policy refuses open;

* wrong environment refuses reopen;
* wrong chain_id refuses reopen;
* wrong genesis/domain refuses reopen;
* wrong namespace refuses reopen;

* missing metadata in non-empty DB refuses open;
* corrupted metadata refuses open;
* unwritable path refuses open/write where representable;

C. Idempotency / equivocation release checks

* duplicate identical `Observed` is idempotent;
* duplicate identical `Observed` after reopen is idempotent;
* duplicate identical `Consumed` is idempotent;
* duplicate identical `Consumed` after reopen is idempotent;
* same record id with different replay sequence fails closed as equivocation;
* same record id with different payload fails closed as equivocation;
* equivocation attempt does not overwrite original;
* original survives reopen after equivocation attempt.

D. Ordering / replay release checks

* `Consumed` before `Observed` fails closed as ordering violation;
* `Consumed` with wrong prior-stage digest fails closed;
* read missing record returns typed not-found and mutates nothing;
* scan empty DB returns empty;
* partial write residue is detected at open;
* `recover_replay_window()` deterministically rolls back safe partial residue if Run 291 supports that behavior;
* unsafe/unknown partial residue remains fail-closed.

E. Corruption release checks

* mutated record payload fails closed on read/scan;
* stale/mismatched record digest fails closed;
* truncated record fails closed;
* corrupt record does not prevent reading a healthy sibling record if the backend supports isolated healthy reads;
* corrupted schema/domain metadata fails closed;
* corrupted record never causes fallback to in-memory.
