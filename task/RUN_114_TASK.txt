# Exact objective

Wire bundle-signing-key ratification enforcement into the SIGHUP live reload path while preserving the existing Run 074 live reload ordering and fail-closed semantics.

The objective is:

1. Enforce bundle-signing-key ratification before any SIGHUP live reload mutation.
2. Use the existing Run 103/104 verifier, Run 105 sidecar input model, and Run 106 per-environment policy.
3. Make MainNet/TestNet ratification mandatory by default.
4. Preserve DevNet explicit opt-in behavior if still required.
5. Reject missing, malformed, wrong-chain, wrong-environment, unknown-root, transport-root, missing-key-material, malformed-key-material, unsupported-suite, or bad-signature ratification before any live reload mutation.
6. Preserve the existing SIGHUP ordering:

```text
trigger → validate → snapshot current trust → swap live trust → evict sessions → commit sequence
```

7. Ensure ratification rejection happens before:
   - current trust snapshot;
   - live trust swap;
   - session eviction;
   - sequence commit;
   - sequence-file write;
   - root merge.
8. Preserve all existing trust-bundle checks:
   - signed-bundle verification;
   - chain_id;
   - environment;
   - Run 055 sequence anti-rollback;
   - activation-height;
   - activation_epoch;
   - Run 065 minimum-margin;
   - revocation activation;
   - local leaf self-check;
   - issuer-root self-check.
9. Preserve repeated-trigger safety:
   - a rejected SIGHUP must not poison future valid reloads;
   - a valid SIGHUP after a rejected SIGHUP must still apply correctly;
   - repeated invalid SIGHUPs must not mutate state or advance sequence.
10. Do not implement peer-driven live apply.
11. Do not implement signing-key rotation/revocation, authority anti-rollback persistence, KMS/HSM, governance, or validator-set rotation.

Run 114 should make this statement true:

> A running node cannot SIGHUP-apply a trust bundle signed by an unratified bundle-signing key on MainNet/TestNet, and rejection occurs before live trust mutation, session eviction, or sequence commit.
