Required lifecycle actions:
At minimum support or explicitly pin these action classes:

1. `ActivateInitial`
- establishes the first active bundle-signing key for a domain, if no prior marker exists;
- must be domain-bound;
- must not bypass genesis/authority-root checks.

2. `Rotate`
- replaces active signing key A with new active signing key B;
- requires higher authority-domain sequence;
- must name previous key A if the schema supports previous-key binding;
- must not accept same-sequence different digest;
- must not accept lower sequence;
- must not accept wrong authority root;
- old key becomes retired or no-longer-active according to current model.

3. `Retire`
- marks a previous signing key as retired without emergency semantics;
- must not remove audit history;
- must not allow a retired key to sign new active bundles after the retirement boundary.

4. `Revoke`
- marks a signing key revoked;
- revoked key must not sign future trust bundles;
- revoked key must not be reactivated by stale lower-sequence material;
- revocation must be domain-bound.

5. `EmergencyRevoke`
- immediate revocation path;
- must bypass only planned activation delay if the design allows that;
- must not bypass signature, authority-root, genesis, chain, environment, or anti-rollback checks;
- must require higher authority-domain sequence.

Required transition rules:
- Every lifecycle transition must be bound to:
  - environment;
  - chain_id;
  - genesis_hash;
  - authority_root_fingerprint;
  - signing-key suite id;
  - active bundle-signing key fingerprint;
  - authority-domain sequence.
- Authority-domain sequence must be strictly monotonic for non-idempotent transitions.
- Same sequence + same digest may be idempotent.
- Same sequence + different digest must be rejected as equivocation.
- Lower sequence must be rejected as rollback.
- Wrong environment / chain / genesis / authority root must be rejected.
- Non-PQC suite IDs must be rejected.
- Revoked signing keys must not authorize new trust-bundle candidates.
- Retired signing keys must not authorize new trust-bundle candidates outside the explicitly allowed overlap window, if any.
- Emergency revocation must not be replayable after a later rotation/revocation state.
- Local marker bytes must be preserved on rejected transitions.
- No sequence file write or live trust mutation occurs in this validator unless invoked through existing apply surfaces.
