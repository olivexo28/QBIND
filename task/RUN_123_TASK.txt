# Exact objective

Wire authority-marker conflict checks into validation-only trust-bundle surfaces without ever persisting authority marker state from those surfaces.

Validation-only surfaces for Run 123:

1. `--p2p-trust-bundle-reload-check`;
2. local `--p2p-trust-bundle-peer-candidate-check`;
3. live inbound `0x05` peer-candidate validation.

The objective is:

1. For validation-only surfaces that already verify ratification, derive `PersistentAuthorityStateRecord` only from verified ratification.
2. Compare the derived marker against the persisted marker using the Run 117/118/119 helper stack.
3. Reject conflict, corruption, wrong-domain, or invalid marker state.
4. Never persist authority marker state from validation-only surfaces.
5. Preserve all existing non-mutation guarantees for reload-check, local peer-candidate check, and live inbound `0x05`.
6. Preserve Run 088 propagation gating: a marker-conflicting live `0x05` candidate must not rebroadcast.
7. Do not implement snapshot/restore conflict enforcement in this run.
8. Do not implement reset/recovery.
9. Do not invent a fake monotonic authority sequence.
10. Do not implement signing-key rotation/revocation lifecycle.
11. Do not implement peer-driven live apply.

Run 123 should make this statement true:

> A validation-only surface cannot accept or rebroadcast a ratified trust candidate whose authority marker conflicts with the local persisted authority marker, and validation-only paths never persist authority marker state.
