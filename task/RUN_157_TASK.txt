This is source/test fixture tooling only. Release-binary positive TestNet apply evidence is deferred to Run 158.

Add or extend fixture tooling so one invocation can produce a coherent TestNet universe with:

A. TestNet domain
- environment = testnet;
- chain_id = qbind-testnet-v0 / 0x51424e4454535400, matching current project constants;
- canonical TestNet genesis file;
- expected TestNet genesis hash;
- same genesis hash used by:
  - baseline trust bundle;
  - candidate trust bundle;
  - v2 ratification sidecar;
  - peer-candidate envelope;
  - local seeded marker, if any.

B. Transport/authentication material
- one transport root authority;
- V0/V1/V2 leaf certs issued by that root;
- V0/V1/V2 ML-KEM leaf keys;
- material usable by real `target/release/qbind-node` N=3 mutual-auth TestNet topology.

- baseline TestNet trust bundle seq=1 trusted by V1 at startup;
- candidate TestNet trust bundle seq=2 that is a valid successor of baseline;
- same chain/environment/genesis/authority universe;
- candidate adds/removes/updates enough material to prove live trust mutation without breaking transport before apply;
- bundle signatures verify under the same configured bundle-signing authority set.

D. v2 ratification material
- v2 ratification sidecar for candidate seq=2;
- signed under authority accepted by TestNet receiver;
- bound to the same:
  - environment;
  - chain_id;
  - genesis_hash;
  - authority root;
  - active bundle-signing key;
  - candidate digest;
  - authority-domain sequence;
- compatible with Run 130 verifier and Run 132/142/150/152 marker logic.

E. authority marker material
- optional seeded v2 marker seq=1 matching baseline, or whatever Run 153/156 receiver requires before candidate validation;
- if emitted, it must be domain-bound to the same TestNet universe;
- candidate seq=2 must compare as `HigherSequenceAccepted` or equivalent.

F. peer-candidate 0x05 envelope
- valid TestNet v2 peer-candidate envelope for candidate seq=2;
- negative fixtures:
  - lower sequence;
  - same-sequence different digest;
  - bad signature;
  - wrong environment;
  - wrong chain;
  - wrong genesis;
  - ambiguous v1+v2;
  - duplicate candidate;
  - expired candidate if current expiry mechanism supports it.
