Core design requirements:

Implement a source/test production governance execution engine that:

1. Defaults to Disabled / fail-closed.

2. Accepts only explicitly verified governance-proof decisions.

A valid engine accept path must require a Run 299 / Run 300-style accepted on-chain governance proof decision. It must not accept an unverified proof, fixture proof, local operator assertion, peer majority, custody-only evidence, RemoteSigner-only evidence, KMS/HSM-only evidence, or custody-attestation-only evidence.

3. Produces a typed execution intent, not live mutation.

Run 301 may produce a deterministic `GovernanceExecutionIntent`, `AuthorityLifecycleExecutionIntent`, `GovernanceExecutionPlan`, or equivalent. It must not directly apply the intent to live trust state.

Allowed intent classes include source/test representations of:

* authority lifecycle rotation intent;
* authority lifecycle retirement intent;
* authority lifecycle revocation intent;
* emergency revocation intent;
* bundle-signing-key authorization / retirement / revocation intent;
* governance no-op / rejected decision;
* future validator-set rotation request as a typed unsupported/fail-closed outcome only.

Do not implement actual validator-set rotation.

Do not write sequence files.

Do not write authority markers.

Do not call Run 070.

4. Binds the full governance decision tuple.

The engine must bind at least:

* environment;
* chain_id;
* genesis hash;
* authority root fingerprint and suite;
* governance domain id;
* governance epoch;
* proposal id;
* proposal digest;
* proposal outcome;
* lifecycle action;
* candidate v2 digest;
* authority-domain sequence;
* decision id / replay nonce;
* quorum / threshold;
* trusted checkpoint / proof transcript digest from Run 299;
* custody provider class / key handle / signer fingerprint where represented;
* custody attestation transcript digest where represented;
* durable replay record id / digest where represented;
* execution policy id;
* execution engine protocol version;
* execution intent digest.

Wrong value for any bound field must fail closed with a typed error.

5. Uses canonical deterministic bytes.

All execution-intent IDs, transcripts, request IDs, plan digests, and audit digests must use deterministic canonical bytes and domain-separated SHA3-256 or the existing project digest convention.

Do not use Rust `Debug` formatting as canonical bytes.

Do not use wall-clock time as an authorizing input.

Freshness must be explicit: governance epoch, governance height, authority-domain sequence, decision id, replay nonce, or validity window.

6. Enforces ordering.

The engine should model the minimum safe ordering:

selector/policy resolution â†’
input well-formedness â†’
verified on-chain governance proof decision check â†’
replay/freshness check â†’
custody backend evidence check where represented â†’
custody attestation evidence check where represented â†’
durable replay / idempotency check where represented â†’
lifecycle action support check â†’
execution intent construction â†’
typed non-mutating outcome.

Mutation must not occur in Run 301.

Only a typed accepted outcome may authorize a future mutation run.

7. Preserves MainNet refusal.

MainNet must remain refused absent production authority criteria.

For Run 301, no MainNet accept path is expected. A valid source/test DevNet/TestNet proof must not enable MainNet runtime behavior. A MainNet policy may be represented but should fail closed with a typed unavailable/refused outcome unless every production prerequisite is explicitly represented and proven. Since validator-set rotation and full MainNet evidence are still Red, do not produce MainNet-ready outcomes.

8. Validator-set rotation remains unsupported.

If a governance proof asks for validator-set rotation / authority-set synchronization, the engine must classify it as unsupported/fail-closed unless a narrow â€śprepared intent onlyâ€ť type is explicitly documented as non-mutating and not an implementation of rotation. The C4/C5 matrix must keep validator-set rotation Red.

9. Governance execution engine does not equal runtime wiring.

Run 301 must not wire the engine into default binary runtime call sites. It can compose with existing source/test runtime/evaluator surfaces, but no default runtime behavior should change. Release-binary evidence is deferred to Run 302.
