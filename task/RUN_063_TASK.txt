Run 063. Implement and prove root-level local revocation startup self-check.

Primary objective:
If the node’s own local PQC transport delegation certificate was issued by a transport root that is actively revoked in the loaded trust bundle, the node must fail closed during startup before P2P is constructed or started.

Important framing:
This is a narrow startup hardening task.
Do not redesign trust bundles.
Do not redesign KEMTLS.
Do not redesign P2P.
Do not modify consensus or timeout logic.
Do not implement epoch gating.
Do not implement KMS/HSM.
Do not implement bundle-signing-key ratification.
Do not implement fast-sync restore.
Do not weaken Run 061 local leaf-fingerprint revocation.
Do not weaken Run 062 pending/active revocation activation-height semantics.
Do not weaken peer-handshake revocation behavior.
Do not silently fall back to --p2p-trusted-root.
Do not silently fall back to DummySig, DummyKem, or DummyAead.
Do not claim full C4 or C5 closure.

Strict scope:
- Decode the local --p2p-leaf-cert.
- Extract the issuing transport root_id from that local NetworkDelegationCert.
- Check the issuing root_id against the active revoked-root set from the loaded trust bundle.
- Use active revoked roots only; pending root revocations from Run 062 must not trigger startup failure.
- Perform the check after:
  - signed bundle verification
  - environment validation
  - chain_id validation
  - bundle/root validity-window validation
  - activation-height validation
  - sequence anti-rollback validation
  - revocation activation filtering
- Perform the check before:
  - pqc_config construction if feasible
  - P2pNodeBuilder starts
  - any peer connection attempt
  - any accepted cert counter could move
- Fail closed with a precise FATAL/log reason.
- Preserve Run 061 local leaf fingerprint self-check.
- Preserve Run 052 peer handshake revoked-leaf behavior.
- Preserve Run 062 pending root revocation behavior.
- Preserve Run 050–062 behavior.
- Crosscheck all code against existing design/spec. If contradictions or inconsistencies are found, update docs/whitepaper/contradiction.md.
