# Exact objective

Produce release-binary evidence proving that the already-wired mutating surfaces persist and enforce the authority anti-rollback marker correctly.

Run 122 should prefer **no production runtime code changes**.

Allowed changes are limited to:

- release-binary evidence scripts;
- evidence-only fixture/helper generation;
- log/metrics/assertion helpers;
- documentation/evidence updates;
- narrow bug fixes if release-binary evidence exposes a real defect in Runs 119–121.

The objective is to prove on real `target/release/qbind-node` binaries:

1. First accepted ratified mutation persists the authority marker.
2. Re-running with the same marker is idempotent.
3. A conflicting marker is rejected before mutation.
4. A corrupt marker fails closed before mutation.
5. Marker is not written from DevNet no-opt-in unratified paths.
6. Marker persistence remains after the existing commit boundary.
7. No validation-only surface persists a marker.
8. No fake authority sequence is introduced.
9. No peer-driven live apply is introduced.

Preferred coverage order:

1. process-start reload-apply, because Run 119 already has the cleanest explicit commit boundary;
2. SIGHUP live reload, because Run 121 added repeated-trigger semantics;
3. startup `--p2p-trust-bundle`, if release-binary setup remains tractable.

If all three are feasible, cover all three. If not, return the honest verdict based on covered surfaces.
