Required scenario matrix:

A1. valid v2 candidate accepted validation-only
- Send a valid v2 peer-candidate over live 0x05.
- Expected:
  - receiver logs v2 path selected;
  - Run 130 verifier succeeds;
  - Run 132/142 v2 validation-only marker check accepts;
  - receiver records accepted/staged/validated outcome;
  - no sequence write on any node;
  - no authority-marker write on any node;
  - no LivePqcTrustState mutation;
  - no session eviction;
  - no peer-driven apply.

A2. idempotent v2 candidate accepted validation-only
- Receiver has matching local v2 marker.
- Send same-sequence/same-digest v2 candidate over live 0x05.
- Expected:
  - accepted/idempotent validation-only;
  - marker bytes unchanged;
  - sequence bytes unchanged;
  - no mutation.

A3. higher-sequence v2 candidate accepted validation-only
- Receiver has local v2 marker seq=N.
- Candidate seq=N+1.
- Expected:

  - no marker persist;
  - no sequence persist;
  - no live trust mutation.

A4. v2-after-v1 migration candidate accepted validation-only
- Receiver has local v1 marker.
- Candidate is valid v2 higher-sequence candidate.
- Expected:

  - existing v1 marker bytes preserved;
  - no v2 marker write;
  - no sequence write.
