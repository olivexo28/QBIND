# Exact objective

Produce release-binary evidence proving that canonical `meta:current_epoch` is consumed correctly by trust-bundle activation surfaces, and that all fail-closed boundaries remain intact.

Run 099 should prefer no runtime code change.

Allowed code changes are limited to:

- test harnesses
- DevNet scripts
- log/evidence helpers
- documentation corrections
- narrowly scoped bug fixes if evidence reveals a real blocker

The objective is to prove:

1. Fresh genesis with no committed epoch still rejects `activation_epoch` with `CurrentEpochUnavailable`.
2. Committed canonical epoch satisfies `activation_epoch <= current_epoch`.
3. Future epoch rejects when `activation_epoch > current_epoch`.
4. Restored epoch-bearing snapshot satisfies `activation_epoch <= restored_epoch`.
5. Old snapshot without epoch remains `CurrentEpochUnavailable`.
6. Peer-candidate validation/propagation rejects unavailable/future epoch candidates and remains non-mutating.
7. No sequence write, root merge, live trust mutation, session eviction, or propagation occurs after rejection.
8. No missing epoch is treated as epoch `0`.
9. No height/time/view/snapshot-height-derived epoch is used.
10. Full C4 and C5 remain open.
