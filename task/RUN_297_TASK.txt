Verifier design requirements:

1. Default Disabled / fail-closed

The attestation verifier policy must default to Disabled.

Disabled must fail closed before any evidence parsing, provider verification, trust-root check, or backend acceptance.

2. Attestation class taxonomy

Model at least:

* Disabled;
* FixtureKmsAttestation;
* FixtureHsmAttestation;
* ProductionCloudKmsAttestation;
* ProductionPkcs11HsmAttestation;
* ProductionGenericKmsAttestation;
* ProductionGenericHsmAttestation;
* RemoteSignerAttestation;
* Unknown / unsupported.

Fixture attestation is allowed only for DevNet/TestNet source tests under explicit fixture policy.

Fixture attestation must be refused for MainNet.

RemoteSigner attestation must not satisfy the KMS/HSM custody attestation row unless explicitly routed through a separate RemoteSigner-specific policy and documented as not KMS/HSM custody.

3. Attestation policy taxonomy

Model at least:

* Disabled;
* FixtureKmsAttestationAllowed;
* FixtureHsmAttestationAllowed;
* ProductionCloudKmsAttestationRequired;
* ProductionPkcs11HsmAttestationRequired;
* ProductionGenericKmsAttestationRequired;
* ProductionGenericHsmAttestationRequired;
* MainnetProductionCustodyAttestationRequired.

Default must be Disabled.

MainNet must require production custody attestation and must refuse fixture/local/RemoteSigner-only/peer-majority evidence.

4. Evidence binding

Every attestation evidence object must bind, where represented:

* environment;
* chain_id;
* genesis hash or authority-domain binding;
* authority root;
* authority-domain sequence;
* custody class;
* provider kind;
* provider identity;
* key handle;
* key fingerprint;
* signer identity;
* request kind;
* authorized action;
* candidate/proposal/request digest;
* custody request id;
* custody request envelope digest;
* custody response envelope digest;
* Run 295 backend transcript digest;
* optional Run 291 durable replay record digest;
* attestation nonce/challenge;
* attestation measurement digest;
* attestation certificate/quote/proof digest;
* attestation protocol version;
* attestation verifier policy;
* domain separation tag.

Wrong environment, chain, genesis/domain, authority sequence, provider kind, provider identity, key handle, signer, authorized action, candidate digest, custody request id, backend transcript, nonce/challenge, measurement, certificate/proof digest, protocol version, or policy must fail closed.

5. Canonical encoding

Use deterministic canonical bytes for all attestation evidence, attestation transcript, and attestation decision digests.

Do not use Rust `Debug` formatting as canonical bytes.

Do not use wall-clock time as an authorizing value.

Freshness must be represented through explicit typed nonce/challenge/sequence fields, not ambient wall-clock acceptance.

* wrong trust root;
* missing key binding;
* wrong key handle;
* wrong provider identity;

* wrong backend transcript;
