Task:
Run 078. Add a disabled-by-default P2P wire receive path for peer-candidate validation only.

Primary objective:
Allow a running qbind-node to receive a bounded trust-bundle peer-candidate envelope over the P2P layer and validate it through the Run 076 PeerCandidateValidator, without applying it, propagating it, persisting sequence, mutating LivePqcTrustState, or evicting sessions.

Important framing:
This is validation-only P2P receive.
This is NOT peer-driven live apply.
This is NOT automatic gossip propagation.
This is NOT trust-bundle synchronization.
This is NOT consensus ratification.
This is NOT activation_epoch.
This is NOT KMS/HSM.
This is NOT signing-key ratification.
This is NOT fast-sync restore.
Do not redesign KEMTLS.
Do not redesign consensus.
Do not weaken local operator reload-check.
Do not weaken local SIGHUP live reload.
Do not silently fall back to --p2p-trusted-root.

Do not claim full C4 or C5 closure.
