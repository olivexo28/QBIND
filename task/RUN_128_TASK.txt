# Exact objective

Produce release-binary evidence proving that the authority-state reset CLI succeeds only in allowed recovery cases and refuses unsafe cases without writing a marker.

Run 128 should prefer **no production runtime code changes**.

Allowed changes are limited to:

- release-binary evidence scripts;
- evidence-only fixture/helper generation;
- log/assertion helpers;
- documentation/evidence updates;
- narrow bug fixes if release-binary evidence exposes a real Run 127 defect.

The objective is to prove on real `target/release/qbind-node` binaries:

1. DevNet reset succeeds with valid inputs, writes target marker, and writes audit record.
2. MainNet local reset refuses by default.
3. Missing ratification refuses.
4. Bad ratification refuses.
5. Wrong expected genesis hash refuses.
6. Wrong environment / wrong chain refuses if feasible.
7. Corrupt existing marker refuses and is not repaired.
8. Missing audit output path refuses.
9. No marker write occurs on every refusal case.
10. Audit record contains stable success/refusal fields.
11. Reset command exits before normal node startup: no P2P, consensus, metrics, SIGHUP, reload, or peer-candidate dispatch markers.
12. No MainNet governance artifact support is implemented or faked.
13. No fake authority sequence is introduced.
14. No signing-key rotation/revocation, KMS/HSM, governance, or peer-driven live apply is introduced.
