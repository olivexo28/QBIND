# Exact objective

Write a production-grade authority and ratification specification for PQC trust anchors and bundle-signing keys.

The objective is to define:

1. How initial trust anchors are established at genesis.
2. How genesis trust-anchor authority is cryptographically bound at boot.
3. How bundle-signing keys are ratified.
4. How bundle-signing keys rotate.
5. How bundle-signing keys are revoked.
6. How DevNet, TestNet, and MainNet differ.
7. What nodes must refuse in each environment.
8. What anti-rollback rules apply to authority state.
9. What persistence and recovery semantics are required before implementation.
10. What future implementation runs must satisfy.

Run 100 must produce a clear design/specification that future implementation runs can follow.

Run 100 should not introduce runtime behavior unless a tiny compile-time/doc-test helper is absolutely necessary. Prefer docs only.

# Architectural decision

Run 100 must use this design direction:

## Initial production authority must come from a genesis configuration file bound by a boot-time cryptographic hash.

Do NOT bake initial production root anchors directly into source code as static constants.

Allowed:

- test-only/dev-only static constants behind explicit non-production gates
- examples/fixtures for tests
- documented DevNet local shortcuts

Not allowed:

- MainNet production root anchors hard-coded in Rust source as authoritative constants
- hidden fallback anchors
- implicit trust roots outside genesis/config
- environment-ambiguous root anchors

The genesis configuration should be the canonical authority root.

Expected model:

```text
genesis config
  ├─ chain_id
  ├─ environment
  ├─ initial validator set
  ├─ initial PQC transport trust anchors
  ├─ initial bundle-signing authority / ratification root
  ├─ initial activation policy parameters
  ├─ initial authority sequence / epoch
  └─ canonical genesis hash

At boot:

load genesis config
→ canonicalize bytes
→ compute genesis hash
→ compare against expected genesis hash
→ verify chain/environment binding
→ derive initial authority state
→ reject any trust-anchor/bundle-signing authority not bound to that genesis
