Design goal:
Extend the governance authority verifier so `GovernanceAuthorityClass::OnChainGovernance` no longer has only a generic unsupported outcome at source/test level. Instead, define a typed OnChainGovernance proof object that can be parsed, domain-bound, and validated against a local mock/fixed governance decision root.

The verifier must remain fail-closed by default on production surfaces unless a valid proof is present and policy explicitly allows OnChainGovernance.

Required proof model:
Define an OnChainGovernance proof type that is explicitly bound to:

* environment;
* chain_id;
* genesis_hash;
* authority_root_fingerprint;
* governance_domain_id;
* governance_epoch or governance_height;
* proposal_id;
* proposal_digest;
* lifecycle action;
* candidate authority bundle digest;
* authority-domain sequence;
* active / new / revoked bundle-signing key fingerprint where applicable;
* decision outcome;
* quorum / threshold metadata;
* proof freshness window;
* replay-protection nonce or unique decision id;
* proof suite id;
* proof bytes or mock proof commitment.

Since real on-chain governance execution is out of scope, the proof may use a deterministic mock verifier or local fixture verifier, but it must be typed as fixture-only and must not be confused with MainNet production proof.

Add a pure verifier, or extend the existing governance verifier, to return typed outcomes such as:
