Objective:
Add a source/test-only governance ratification authority verifier for v2 bundle-signing-key lifecycle transitions.

This verifier must define and validate the local proof object that, in a future run, can authorize MainNet/TestNet governance-controlled bundle-signing-key lifecycle transitions. It must not enable MainNet apply or mutate any live trust state.

This run is non-applying and source/test only.

- Source/test only.

- No MainNet peer-driven apply enablement.
- No governance execution engine.

- No KMS/HSM implementation.
- No validator-set rotation.
- No autonomous apply.
- No automatic apply on receipt.

Required design:
Add a typed governance authority verifier module, preferably:

crates/qbind-node/src/pqc_governance_authority.rs

The verifier must be pure and non-mutating.

It should validate a governance authority proof for a v2 lifecycle transition against:

- environment;
- chain_id;
- genesis_hash;
- authority root fingerprint;
- lifecycle action;
- active / new / revoked bundle-signing key fingerprint;
- authority-domain sequence;
- candidate digest;
- issuer authority class;
- issuer signature suite;
- issuer signature;
- governance threshold metadata if already representable.

Required trust model:
Define a typed model for at least:

1. GenesisBound authority
- proof chains to genesis-bound bundle-signing authority root;
- valid for DevNet/TestNet fixtures;
- future MainNet-compatible but does not enable MainNet apply.

2. EmergencyCouncil authority
- proof represents emergency revocation authority;
- must be domain-bound;
- must not bypass signature, genesis, chain, environment, or sequence checks.

3. OnChainGovernance authority
- placeholder verifier may reject as unsupported if no on-chain proof format exists yet;
- must be explicitly fail-closed.

If the current schema cannot represent these classes, document the exact schema gap and implement only the representable subset as source/test with typed `UnsupportedProofShape` results.

Required verifier behavior:
The verifier must return typed outcomes, for example:

GovernanceAuthorityVerificationOutcome {
    AcceptedGenesisBound { ... },
    AcceptedEmergencyCouncil { ... },
    UnsupportedOnChainGovernance,
    WrongEnvironment,
    WrongChain,
    WrongGenesis,
    WrongAuthorityRoot,
    WrongLifecycleAction,
    WrongCandidateDigest,
    WrongAuthoritySequence,
    InvalidIssuerSignature,
    UnsupportedIssuerSuite,
    NonPqcSuiteRejected,
    ThresholdNotMet,
    MalformedProof,
    ReplayRejected,
}

Names can differ, but outcomes must be typed and testable.

Required accepted cases:
A1. Genesis-bound Rotate proof accepted
- valid proof;
- correct environment/chain/genesis/root;
- correct lifecycle action;
- correct candidate digest;
- correct authority-domain sequence;
- PQC issuer suite;
- signature verifies.

A2. Genesis-bound Revoke proof accepted
- same as A1 but revoke action.

A3. Genesis-bound EmergencyRevoke proof accepted if the model allows genesis-bound emergency revocation.

A4. EmergencyCouncil EmergencyRevoke proof accepted if representable.

A5. Idempotent same governance proof accepted / classified as idempotent if replay-safe.

Required rejection cases:
R1. wrong environment rejected.

R2. wrong chain rejected.

R3. wrong genesis rejected.

R4. wrong authority root rejected.

R5. wrong lifecycle action rejected.

R6. wrong candidate digest rejected.

R7. wrong authority-domain sequence rejected.

R8. invalid issuer signature rejected.

R9. unsupported issuer suite rejected.

R10. non-PQC suite rejected.

R11. threshold not met rejected, if threshold metadata is representable.

R12. malformed proof rejected.

R13. stale/replayed lower authority proof rejected.

R14. OnChainGovernance proof rejected as unsupported if no proof format exists.

R15. local operator config alone rejected as MainNet authority proof.

R16. peer majority / gossip count rejected as authority proof.

Required integration boundary:
Do not wire this verifier into mutating apply surfaces yet unless the task explicitly proves it is safe.

Instead, add a pure integration helper that composes:

v2 lifecycle transition
+ governance authority proof
+ trust domain

into a single non-mutating decision, for example:

validate_lifecycle_with_governance_authority(...)

This helper must:
- call Run 159 lifecycle validation;
- call the new governance authority verifier;
- return a typed combined decision;
- perform no I/O;
- write no marker;
- write no sequence;
- mutate no live trust state.
