# Exact objective

Wire the Run 117 authority anti-rollback marker into ratification-enforced surfaces, starting with mutating surfaces where rollback risk is highest.

The objective is:

1. Before accepting a ratified bundle-signing authority state, derive a `PersistentAuthorityStateRecord` from the verified ratification context.
2. Compare the derived marker against the persisted marker from Run 117.
3. Reject rollback/conflict/corrupt/wrong-chain/wrong-environment/wrong-genesis authority state before any trust mutation.
4. Persist the marker atomically when a mutating surface successfully accepts a ratified authority state that should become durable.
5. Keep the marker distinct from Run 055 trust-bundle sequence, `activation_height`, `activation_epoch`, and Run 065 minimum-margin.
6. Do not claim full signing-key rotation safety because the current ratification object lacks a monotonic per-key sequence/epoch.
7. Do not implement signing-key rotation/revocation lifecycle.
8. Do not implement peer-driven live apply.

Run 118 should make this statement true for the first wired surfaces:

> Ratification-enforced trust-bundle acceptance cannot silently roll back to a conflicting or stale ratified authority marker; conflicts fail closed before mutation, and accepted mutating operations persist the marker atomically.
