## Current state

QBIND is a post-quantum-secure L1 blockchain. Security, correctness, production safety, and evidence discipline take priority over speed or convenience.

Recent state:

  - startup `--p2p-trust-bundle` preflight gate before Run 055 sequence write and root merge;

  - sidecar JSON ratification input loader;
  - hidden local-only CLI flags;
  - typed `ReloadCheckError::RatificationRefused`;
  - no peer-candidate-check enforcement yet.
- Run 106 added default-strict ratification policy for the two already-wired surfaces:
  - MainNet/TestNet invoke ratification regardless of opt-in flag;
  - DevNet preserves explicit operator opt-in;
  - startup and reload-check now consult `ratification_gate_decision(env, opt_in)`;
  - peer-candidate-check, live peer-candidate wire validation, propagation, reload-apply, and SIGHUP remained deferred.

Run 106 was partial-positive because only 2 of 7 target surfaces are default-strict. The next safest surface is the local peer-candidate check CLI because it is mechanically closest to reload-check and remains validation-only.

Run 107 must wire ratification enforcement into:

```text
--p2p-trust-bundle-peer-candidate-check
```

This run must not touch live peer-candidate wire validation, propagation/rebroadcast, reload-apply, SIGHUP, peer-driven live apply, rotation/revocation, authority anti-rollback persistence, KMS/HSM, or governance.

# Exact objective

Implement ratification enforcement for the peer-candidate check CLI while preserving its validation-only, non-mutating behavior.

The objective is:

1. Reuse the Run 105 / Run 106 ratification policy and sidecar input model.
2. Wire `--p2p-trust-bundle-peer-candidate-check` into ratification-aware validation.
3. Make MainNet/TestNet strict by default for peer-candidate check.
4. Preserve DevNet opt-in behavior exactly and explicitly.
5. Reject unratified, missing, malformed, wrong-chain, wrong-environment, bad-signature, unknown-root, transport-root, unsupported-suite, and missing-key-material ratification cases before validation success.
6. Preserve every Run 076–090 peer-candidate non-mutation invariant:
   - no sequence write;
   - no live trust mutation;
   - no root merge;
   - no session eviction;
   - no propagation;
   - no apply;
   - no consensus ratification.
7. Do not change the peer-candidate wire format.
8. Do not touch live peer-candidate dispatcher behavior in this run.

Run 107 should make this statement true:

> The local `--p2p-trust-bundle-peer-candidate-check` path rejects MainNet/TestNet peer candidates signed by unratified bundle-signing keys before any mutation or propagation side effect.

- print precise error;
- do not write sequence;
- do not mutate trust;

- environment check;
- activation-height check;
- activation_epoch check;
- sequence peek / anti-rollback check;
- revocation checks;
- local-leaf / issuer-root checks;
- size/version/wire envelope checks.

## 5. Tests and evidence helpers

Add helper support only as needed for peer-candidate check evidence.

Do not add production shortcuts.
