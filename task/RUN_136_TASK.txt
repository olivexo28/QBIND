# Exact objective

Wire ratification v2 verification and v2 authority-marker compatibility into the startup `--p2p-trust-bundle` mutating surface only.

The objective is:

1. Extend startup `--p2p-trust-bundle` to accept either v1 or v2 ratification sidecars through the existing versioned dispatch model.
2. Preserve existing v1 startup behavior exactly.
3. For v2 sidecars, verify using Run 130 `verify_bundle_signing_key_ratification_v2`.
4. For v2 sidecars, derive v2 marker using Run 131 helpers.
5. Compare v2 marker against the persisted authority marker before any startup trust mutation or network start.
6. Reject unsafe v2 cases before Run 055 sequence/root mutation, bundle-root merge, or P2P/network startup.
7. Persist the v2 marker only after the accepted startup commit boundary.
8. Enforce v1/v2 compatibility policy:
   - v2-after-v1 is explicit migration and may persist V2 marker only after accepted startup commit;
   - v1-after-v2 fails closed;
   - lower v2 sequence fails closed;
   - same-sequence/different-digest fails closed;
   - same-sequence/same-digest is idempotent;
   - higher sequence is accepted only if verified and marker comparison allows it.
9. Preserve DevNet no-opt-in behavior: if ratification is skipped by policy, do not write v2 marker from unratified state.
10. Do not wire SIGHUP, live `0x05`, or other mutating surfaces.

Run 136 should make this statement true:

> Startup `--p2p-trust-bundle` can safely consume v2 ratifications and persist v2 authority marker state only after the existing startup commit boundary, while rejecting downgrade/replay/equivocation before startup mutation or network start and preserving all v1 behavior.
