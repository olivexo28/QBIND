# Exact objective

Wire ratification v2 verification and v2 authority-marker compatibility into the process-start reload-apply mutating surface only.

The objective is:

1. Extend process-start reload-apply to accept either v1 or v2 ratification sidecars through the existing versioned dispatch model.
2. Preserve existing v1 reload-apply behavior exactly.
3. For v2 sidecars, verify using Run 130 `verify_bundle_signing_key_ratification_v2`.
4. For v2 sidecars, derive v2 marker using Run 131 helpers.
5. Compare v2 marker against the persisted authority marker before any reload-apply mutation.
6. Reject unsafe v2 cases before `snapshot → swap → evict_sessions → commit_sequence`.
7. Persist the v2 marker only after `commit_sequence` succeeds.
8. Enforce v1/v2 compatibility policy:
   - v2-after-v1 is explicit migration and may persist v2 marker only after successful reload-apply commit;
   - v1-after-v2 fails closed;
   - lower v2 sequence fails closed;
   - same-sequence/different-digest fails closed;
   - same-sequence/same-digest is idempotent;
   - higher sequence is accepted only if the verified v2 ratification and marker comparison allow it.
9. Preserve no-marker-write-on-failed-apply behavior.
10. Preserve no-marker-write-on-rejected-v2 behavior.
11. Do not wire startup, SIGHUP, live `0x05`, or other mutating surfaces.

Run 134 should make this statement true:

> Process-start reload-apply can safely consume v2 ratifications and persist v2 authority marker state only after the existing commit boundary, while rejecting downgrade/replay/equivocation cases before mutation and preserving all v1 behavior.

## D. Compatibility policy

- no marker + v2 candidate: accept if ratification and marker comparison allow, persist after commit;
- v1 marker + v2 candidate: explicit migration candidate, persist v2 marker after commit;
- v2 marker + v1 candidate: reject before mutation;
- v2 marker + lower v2 sequence: reject before mutation;
- v2 marker + same sequence / same digest: idempotent, no rewrite unless existing convention requires;
- v2 marker + same sequence / different digest: reject before mutation;
- v2 marker + higher sequence: accept only if verified and comparison allows, persist after commit.

## E. Existing ordering preservation

Preserve Run 070/073/119 ordering:

```text
validate/ratify → marker compare → snapshot → swap → evict_sessions → commit_sequence → persist_authority_marker
```

Do not move marker persistence before `commit_sequence`.
