# Exact objective

Wire v2 ratification verification and v2 authority-marker compatibility checks into the safest validation-only surfaces first.

Primary surfaces for Run 132:

1. `--p2p-trust-bundle-reload-check`;
2. local `--p2p-trust-bundle-peer-candidate-check`.

Optional only if clearly narrow after primary surfaces:

3. live inbound `0x05` validation-only path.

The objective is:

1. Allow validation-only surfaces to verify either v1 or v2 ratification sidecars according to a strict compatibility policy.
2. Preserve existing v1 behavior bit-for-bit when a v1 sidecar is used.
3. For v2 sidecars, use `verify_bundle_signing_key_ratification_v2` from Run 130.
4. For v2 sidecars, derive and compare v2 authority marker state using Run 131 helpers.
5. Enforce the Run 129/131 compatibility policy:
   - v2-after-v1 may be accepted only as explicit migration candidate;
   - v1-after-v2 must fail closed;
   - lower v2 sequence must fail closed;
   - same-sequence/different-digest must fail closed;
   - same-sequence/same-digest is idempotent;
   - higher v2 sequence may be accepted by validation-only check but never persisted by validation-only surfaces.
6. Never persist marker state from validation-only surfaces.
7. Do not wire v2 into startup, reload-apply, or SIGHUP mutating surfaces yet.
8. Do not implement signing-key rotation/revocation lifecycle yet.
9. Do not add peer-driven apply.

Run 132 should make this statement true:

> QBIND validation-only trust-bundle checks can safely evaluate v2 ratifications and v2 monotonic marker compatibility without mutating local authority state, while preserving v1 fail-closed behavior and refusing unsafe downgrade/replay cases.
