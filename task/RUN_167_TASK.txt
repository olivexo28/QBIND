Required design:
Add an additive, versioned governance-proof carrier to the v2 authority/ratiﬁcation sidecar path.

Preferred shape:
- Extend the existing v2 ratification sidecar structure with an optional field such as:
  - `governance_authority_proof: Option<GovernanceAuthorityProofWire>`
  - or equivalent.
- Existing sidecars without this field must remain parseable and valid under `GovernanceProofPolicy::NotRequired`.
- Existing sidecars without this field must fail closed under `GovernanceProofPolicy::RequiredForLifecycleSensitive`.
- Sidecars with malformed governance-proof data must fail closed.
- Sidecars with valid governance-proof data must construct `GovernanceProofContext::Available(...)` or equivalent and pass it into the Run 165 governance gate.

Canonical binding requirements:
The governance proof must be bound to:
- environment;
- chain_id;
- genesis_hash;
- authority root fingerprint;
- lifecycle action;
- candidate digest;
- authority-domain sequence;
- active / new / revoked bundle-signing key fingerprint where applicable;
- issuer authority class;
- issuer signature suite;
- issuer signature.

The proof must not be interpreted as:
- peer-majority authority;
- local operator config;
- MainNet apply authorization by itself;
- KMS/HSM custody;
- on-chain governance execution proof.

OnChainGovernance remains unsupported/fail-closed unless a real proof format is implemented, which is out of scope for this run.

Required source behavior:
1. Define a wire-safe governance proof representation, preferably near the existing v2 sidecar types.
2. Add conversion between the wire representation and `GovernanceAuthorityProof`.
3. Add parsing / serialization tests.
4. Update the existing v2 sidecar loader so it can return:
   - `GovernanceProofContext::Unavailable` when no proof is present;
   - `GovernanceProofContext::Available(proof)` when proof parses;
   - typed parse error when proof is malformed.
5. Update production source preflight contexts to pass the parsed governance proof context into:
   - reload-check;
   - local peer-candidate-check;
   - reload-apply;
   - startup `--p2p-trust-bundle`;
   - SIGHUP live reload;
   - live inbound `0x05`;
   - peer-driven drain / `ProductionV2MarkerCoordinator`,
   wherever these surfaces already consume v2 sidecar/context data.
6. Preserve existing `NotRequired` compatibility behavior.
7. Preserve existing `RequiredButMissing` behavior for proof-required candidates with no proof.
8. Do not perform any marker write, sequence write, live trust swap, or session eviction during proof parsing.
