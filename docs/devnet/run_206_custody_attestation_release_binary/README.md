# Run 206 — Release-binary custody attestation verifier boundary evidence

## Scope

Closes the Run 205-deferred release-binary boundary for the source/test
production **custody attestation verifier skeleton** added by
[`crates/qbind-node/src/pqc_custody_attestation_verifier.rs`](
  ../../../crates/qbind-node/src/pqc_custody_attestation_verifier.rs).
Run 205 added the typed attestation verifier on top of the Run 188 typed
authority-custody boundary
[`crates/qbind-node/src/pqc_authority_custody.rs`](
  ../../../crates/qbind-node/src/pqc_authority_custody.rs),
the Run 203 KMS/HSM backend boundary
[`crates/qbind-node/src/pqc_authority_kms_hsm_backend.rs`](
  ../../../crates/qbind-node/src/pqc_authority_kms_hsm_backend.rs),
and the Run 201 RemoteSigner transport boundary:

* the attestation class / policy taxonomy `CustodyAttestationClass`
  (9 variants) / `CustodyAttestationPolicy` (7 variants: disabled,
  fixture-allowed, production-required, mainnet-production-required, and
  the cloud-KMS / PKCS#11 / HSM / RemoteSigner / production unavailable
  classes);
* the `CustodyAttestationEvidence` / `CustodyAttestationInput` structs
  (authority root / signing-key fingerprints, environment / chain /
  genesis, custody class, backend/signer id, key id, lifecycle action,
  candidate digest, authority-domain sequence, nonces, freshness/expiry,
  commitment sentinels);
* deterministic, domain-separated evidence / input / transcript /
  provider-identity digests (`evidence_digest`, `input_digest`,
  `attestation_transcript_digest`, `provider_identity_digest`);
* the pure/mockable `CustodyAttestationVerifier` trait with the
  DevNet/TestNet source/test-only `FixtureCustodyAttestationVerifier`
  and the fail-closed `ProductionAttestationVerifier` /
  `CloudKmsAttestationVerifier` / `Pkcs11HsmAttestationVerifier` /
  `HsmAttestationVerifier` / `RemoteSignerAttestationVerifier`;
* the typed `CustodyAttestationOutcome` taxonomy;
* the pure `verify_custody_attestation`, the composition
  `validate_custody_metadata_and_attestation`, and its alias
  `validate_lifecycle_custody_and_attestation`.

Run 205 is source/test only with the A1–A14 / R1–R40 corpus
[`crates/qbind-node/tests/run_205_custody_attestation_verifier_tests.rs`](
  ../../../crates/qbind-node/tests/run_205_custody_attestation_verifier_tests.rs).
**Run 206 is the release-binary evidence run** for that surface: it
proves the attestation verifier boundary on the **real**
`target/release/qbind-node` plus a release-built helper linking the
production library symbols.

## What this run proves

1. fixture custody attestation remains DevNet/TestNet evidence-only;
2. production / cloud-KMS / PKCS#11 / HSM / RemoteSigner attestation
   verifiers remain unavailable / fail-closed;
3. evidence, input, transcript, and provider-identity digests are
   deterministic and domain-bound;
4. the attestation verifier composes with the Run 188 custody metadata;
5. the attestation verifier composes with the Run 203 KMS/HSM backend and
   Run 201 RemoteSigner transport boundaries where feasible;
6. rejected attestation cases produce no mutation;
7. MainNet peer-driven apply remains refused;
8. no real KMS / HSM attestation / RemoteSigner backend / governance
   execution / validator-set rotation is claimed.

## Honest limitations

* No real cloud KMS / PKCS#11 / HSM vendor attestation verifier is
  implemented. The `ProductionAttestationVerifier` /
  `CloudKmsAttestationVerifier` / `Pkcs11HsmAttestationVerifier` /
  `HsmAttestationVerifier` / `RemoteSignerAttestationVerifier` always fail
  closed as typed unavailable outcomes.
* Fixture custody attestation remains DevNet/TestNet evidence-only and is
  refused on a MainNet trust domain.
* No real RemoteSigner backend / networked signer daemon is implemented;
  RemoteSigner and KMS/HSM remain backend-boundary only.
* governance execution remains unimplemented; real on-chain proof
  verification remains unimplemented; validator-set rotation remains open.
* MainNet peer-driven apply remains refused (Run 147 FATAL invariant).
* Full C4 remains open. C5 remains open.
