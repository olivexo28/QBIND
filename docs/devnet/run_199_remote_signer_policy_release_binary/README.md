# Run 199 — Release-binary RemoteSigner policy selector evidence

## Scope

Closes the Run 198-deferred release-binary boundary for the source/test
hidden **RemoteSigner policy selector** added by
[`crates/qbind-node/src/pqc_remote_signer_policy_surface.rs`](
  ../../../crates/qbind-node/src/pqc_remote_signer_policy_surface.rs).
Run 198 added the typed selector surface on top of the Run 196
RemoteSigner attestation payload-carrying surface
[`crates/qbind-node/src/pqc_remote_signer_payload_carrying.rs`](
  ../../../crates/qbind-node/src/pqc_remote_signer_payload_carrying.rs)
and the Run 194 typed RemoteSigner production-custody boundary
[`crates/qbind-node/src/pqc_remote_authority_signer.rs`](
  ../../../crates/qbind-node/src/pqc_remote_authority_signer.rs):

* the hidden env-var name
  `QBIND_P2P_TRUST_BUNDLE_REMOTE_SIGNER_POLICY` and the hidden clap flag
  `--p2p-trust-bundle-remote-signer-policy` (`hide = true`);
* the typed `RemoteSignerPolicySelectorParseError`
  (`Empty` / `UnknownValue`);
* the pure parsers `remote_signer_policy_from_selector` and
  `remote_signer_policy_env_selector`, plus the CLI/env resolver
  `remote_signer_policy_from_cli_or_env` (CLI-over-env precedence;
  default `RemoteSignerPolicy::Disabled`; invalid values fail closed);
* the seven per-surface preflight wrappers
  `preflight_v2_marker_remote_signer_for_{reload_check, reload_apply,
  startup_p2p_trust_bundle, sighup, local_peer_candidate_check,
  live_inbound_0x05, peer_driven_drain}` that bind the resolved
  `RemoteSignerPolicy` into the Run 196 `RemoteSignerCallsiteContext`
  used by the matching Run 196 routing helper.

Run 198 is source/test only with the selector + A1–A11 / R1–R34 corpus
[`crates/qbind-node/tests/run_198_remote_signer_policy_selector_tests.rs`](
  ../../../crates/qbind-node/tests/run_198_remote_signer_policy_selector_tests.rs).
**Run 199 is the release-binary evidence run** for that surface: it
proves the selector resolves and routes the selected policy into the
seven production preflight contexts on the **real**
`target/release/qbind-node` and a release-built helper linking the
production library symbols.

## What this run proves

1. default behaviour (no CLI, no env) remains `RemoteSignerPolicy::Disabled`;
2. the hidden CLI selector resolves;
3. the env selector resolves;
4. CLI-over-env precedence is deterministic;
5. invalid selector values fail closed with typed parse errors;
6. the resolved policy reaches all seven production preflight contexts;
7. fixture loopback RemoteSigner material passes only where the selected
   policy allows (DevNet/TestNet `FixtureLoopbackAllowed`);
8. production RemoteSigner remains fail-closed as unavailable;
9. MainNet peer-driven apply remains refused even with
   `mainnet-production-remote-signer-required` and fixture loopback
   material;
10. no real RemoteSigner / KMS / HSM / governance-execution /
    validator-set-rotation is claimed.

## Honest limitations

* No real RemoteSigner backend / networked signer service is implemented.
  Every `Production` signer-mode response fails closed as
  `ProductionRemoteSignerUnavailable` / `MainNetProductionRemoteSignerUnavailable`.
* Fixture loopback RemoteSigner remains DevNet/TestNet evidence-only and
  cannot satisfy MainNet production RemoteSigner.
* No real KMS / HSM / cloud-KMS / PKCS#11 backend is implemented.
* governance execution remains unimplemented; real on-chain proof
  verification remains unimplemented; validator-set rotation remains open.
* MainNet peer-driven apply remains refused (Run 147 FATAL invariant).
* Full C4 remains open. C5 remains open.
