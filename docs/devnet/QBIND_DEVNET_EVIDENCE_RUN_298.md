## 6. Attestation class / policy release evidence

The release helper exercises the full attestation-class/policy taxonomy in release mode. `ProductionCustodyAttestationClass` distinguishes fixture KMS/HSM attestation classes from production cloud-KMS / PKCS#11-HSM / generic-KMS / generic-HSM attestation classes. `ProductionCustodyAttestationVerifierPolicy` covers `Disabled` (default, `DisabledNoVerification`), `FixtureKmsAttestationAllowed`, `FixtureHsmAttestationAllowed`, the production-required policies, and `MainnetProductionCustodyRequired`. A fixture attestation under a fixture-allowed policy on DevNet/TestNet verifies (`FixtureKmsAttestationVerified` / `FixtureHsmAttestationVerified`); a fixture attestation under any production policy fails closed (`FixtureAttestationRejectedForMainNet`); a production attestation verifier stub (`ProductionCustodyAttestationVerifierStub`) is reachable but fails closed as unavailable/misconfigured without a real trust root / quote verifier / certificate chain / hardware proof (`ProductionAttestationUnavailable`, `QuoteVerifierUnavailable`, `VerificationMaterialUnavailable`, `AttestationUnavailable`); a RemoteSigner-shaped attestation is rejected as `RemoteSignerAttestationIsNotKmsHsmCustody` (attestation is not RemoteSigner). Protocol version is pinned to `PRODUCTION_CUSTODY_ATTESTATION_PROTOCOL_VERSION`.

## 7. Evidence/request/response/transcript/domain binding release evidence

The release helper proves, in release mode, that the Run 297 verifier derives deterministic domain-separated digests via the challenge/evidence/trust-root/transcript/decision domain tags (`PRODUCTION_CUSTODY_ATTESTATION_CHALLENGE_DOMAIN_TAG`, `…_EVIDENCE_DOMAIN_TAG`, `…_TRUST_ROOT_DOMAIN_TAG`, `…_TRANSCRIPT_DOMAIN_TAG`, `…_DECISION_DOMAIN_TAG`, `…_DOMAIN_SEPARATION_TAG`). The challenge, evidence, provider-identity, and trust-root digests are stable methods; `production_custody_attestation_transcript_digest(...)` and `production_custody_attestation_decision_digest(&transcript, tag)` bind provider, key handle, custody class, signer, request id, request/response envelopes, measurement, challenge, and trust root. A wrong domain fails closed as `ProductionAttestationDomainMismatch`; a wrong provider/key/signer/custody-class/measurement/trust-root/candidate-digest/request-id/request-envelope/response-envelope/backend-transcript fails closed as the corresponding typed `…Mismatch`. Two identical evidence inputs produce identical transcript/decision digests.

> Name note: the task lists `production_custody_attestation_challenge_digest` and `production_custody_attestation_evidence_digest` as free functions, but in the Run 297 module they are the methods `challenge.challenge_digest()` and `evidence.evidence_digest()`. This substitution is recorded in the helper module doc and here.
