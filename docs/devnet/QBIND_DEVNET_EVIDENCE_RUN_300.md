## 7. Trusted checkpoint / root / inclusion-proof binding release evidence

The release helper proves, in release mode, that the Run 299 verifier binds the decision to environment / chain id / genesis hash / authority-root fingerprint+suite / governance domain / epoch / proposal id / proposal digest / proposal outcome / quorum / threshold / lifecycle action / candidate v2 digest / authority-domain sequence / decision id via deterministic length-prefixed domain-separated SHA3-256 decision and proof digests (`production_onchain_governance_decision_digest`, `production_onchain_governance_proof_digest`, `production_onchain_governance_transcript_digest`, `PRODUCTION_ONCHAIN_GOVERNANCE_DOMAIN_SEPARATION_TAG`). The trusted governance root/checkpoint (`ProductionOnChainGovernanceTrustedCheckpoint`) is supplied explicitly out-of-band through `ProductionOnChainGovernanceVerificationInputs`; the proof can never self-authorize its root. Real SHA3-256 Merkle inclusion (`ProductionOnChainGovernanceInclusionProof`, `build_merkle_inclusion_proof`, `RealMerkleInclusionVerifier`) recomputes the decision-leaf root and requires it to equal both the proof's claimed root and the trusted root; a wrong root, a tampered sibling path, or a wrong claimed root all fail closed. Identical inputs produce identical decision/proof/transcript digests.
