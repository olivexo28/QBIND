# Run 202 — Release-binary RemoteSigner transport boundary evidence

## Scope

Closes the Run 201-deferred release-binary boundary for the source/test
production **RemoteSigner transport boundary** added by
[`crates/qbind-node/src/pqc_remote_signer_transport.rs`](
  ../../../crates/qbind-node/src/pqc_remote_signer_transport.rs).
Run 201 added the typed transport boundary on top of the Run 194 typed
RemoteSigner production-custody boundary
[`crates/qbind-node/src/pqc_remote_authority_signer.rs`](
  ../../../crates/qbind-node/src/pqc_remote_authority_signer.rs):

* the transport identity/endpoint config `RemoteSignerTransportConfig`
  (endpoint URI, `signer_id`, `custody_key_id`, authority root /
  signing-key fingerprints, environment / chain / genesis, `suite_id`,
  expected signer-identity digest, optional transport-attestation
  digest, and a `TransportTimeoutRetryPolicy`);
* the `RemoteSignerTransportRequestEnvelope` /
  `RemoteSignerTransportResponseEnvelope` that wrap and bind the Run 194
  `RemoteSignerRequest` / `RemoteSignerResponse`;
* deterministic, domain-separated request / response / transcript
  digests (`envelope_digest` on both envelopes,
  `transport_transcript_digest`, and
  `remote_signer_response_canonical_digest`);
* the pure/mockable `RemoteSignerTransport` trait with the DevNet/TestNet
  source/test-only `FixtureLoopbackRemoteSignerTransport` and the
  fail-closed `ProductionRemoteSignerTransport`;
* the typed `RemoteSignerTransportOutcome` taxonomy;
* the verifier `validate_remote_signer_transport`, the custody-class
  router `validate_remote_signer_transport_for_custody_class`, and the
  composition `validate_lifecycle_custody_remote_signer_and_transport`.

Run 201 is source/test only with the A1–A10 / R1–R35 corpus
[`crates/qbind-node/tests/run_201_remote_signer_transport_boundary_tests.rs`](
  ../../../crates/qbind-node/tests/run_201_remote_signer_transport_boundary_tests.rs).
**Run 202 is the release-binary evidence run** for that surface: it
proves the transport boundary on the **real** `target/release/qbind-node`
plus a release-built helper linking the production library symbols.

## What this run proves

1. fixture loopback transport remains DevNet/TestNet evidence-only;
2. production transport remains unavailable / fail-closed;
3. MainNet production transport remains unavailable / fail-closed;
4. request, response, and transcript digests are deterministic and
   domain-bound;
5. the transport composes with the Run 194 RemoteSigner
   request/response;
6. the transport composes with the custody/RemoteSigner validation path
   at release-helper / library level;
7. MainNet peer-driven apply remains refused;
8. no real RemoteSigner backend / networked signer daemon / KMS / HSM /
   governance execution / validator-set rotation is claimed.

## Reproduce

```bash
bash scripts/devnet/run_202_remote_signer_transport_release_binary.sh
```

The harness is idempotent: it wipes and regenerates everything under this
directory except `README.md`, `summary.txt`, and `.gitignore`.

## Honest limitations

* No real RemoteSigner backend / networked signer daemon is implemented.
  The `ProductionRemoteSignerTransport` always fails closed as
  `ProductionTransportUnavailable` /
  `MainNetProductionTransportUnavailable`.
* Fixture loopback transport remains DevNet/TestNet evidence-only and is
  refused on a MainNet trust domain.
* No real KMS / HSM / cloud-KMS / PKCS#11 backend is implemented.
* governance execution remains unimplemented; real on-chain proof
  verification remains unimplemented; validator-set rotation remains open.
* MainNet peer-driven apply remains refused (Run 147 FATAL invariant).
* Full C4 remains open. C5 remains open.
