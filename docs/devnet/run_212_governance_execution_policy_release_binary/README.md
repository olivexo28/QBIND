# Run 212 — Release-binary governance execution policy-boundary evidence

## Scope

Closes the Run 211-deferred release-binary boundary for the source/test
governance **execution policy boundary** added by
[`crates/qbind-node/src/pqc_governance_execution_policy.rs`](
  ../../../crates/qbind-node/src/pqc_governance_execution_policy.rs).
Run 211 added, at the source/test level only, a typed model describing how an
*approved governance decision* authorizes an authority lifecycle action:

* the typed [`GovernanceExecutionClass`](
    ../../../crates/qbind-node/src/pqc_governance_execution_policy.rs)
  (`Disabled`, `FixtureGovernance`, `EmergencyCouncilFixture`,
  `OnChainGovernanceUnavailable`, `ProductionGovernanceUnavailable`,
  `MainnetGovernanceUnavailable`, `Unknown`) and the typed
  [`GovernanceExecutionPolicy`](
    ../../../crates/qbind-node/src/pqc_governance_execution_policy.rs)
  (`Disabled` default, `FixtureGovernanceAllowed`,
  `EmergencyCouncilFixtureAllowed`, `ProductionGovernanceRequired`,
  `MainnetGovernanceRequired`);
* the typed [`GovernanceAction`] (initial activation / rotate / retire /
  revoke / emergency-revoke, the four policy-change placeholders, the
  validator-set-rotation placeholder, and `Unknown`) and the
  [`GovernanceQuorumThreshold`] metadata;
* the typed [`GovernanceExecutionInput`], [`GovernanceExecutionDecision`], and
  [`GovernanceExecutionExpectations`] structs;
* the deterministic, domain-separated input / decision / transcript / policy
  digest helpers (`input_digest`, `decision_digest`,
  `governance_execution_transcript_digest`,
  `governance_execution_policy_digest`);
* the pure / mockable [`GovernanceExecutionEvaluator`] trait, the DevNet/TestNet
  [`FixtureGovernanceExecutionEvaluator`], and the production / on-chain /
  MainNet evaluators that are callable but fail closed as unavailable;
* the pure [`evaluate_governance_execution_policy`] and the typed
  [`GovernanceExecutionOutcome`];
* the peer-driven guard composition
  (`evaluate_governance_execution_with_peer_driven_guard`,
  [`GovernanceExecutionComposedOutcome`]) and the fail-closed helpers
  (`mainnet_peer_driven_apply_remains_refused_under_governance_execution`,
  `local_operator_cannot_satisfy_governance_execution`,
  `peer_majority_cannot_satisfy_governance_execution`,
  `validator_set_rotation_remains_unsupported`).

Run 211 is source/test only with the corpus
[`crates/qbind-node/tests/run_211_governance_execution_policy_tests.rs`](
  ../../../crates/qbind-node/tests/run_211_governance_execution_policy_tests.rs).
**Run 212 is the release-binary evidence run** for that boundary: it proves the
governance-execution policy boundary on the **real** `target/release/qbind-node`
plus a release-built helper linking the production library symbols.

## What this run proves

1. fixture governance execution accepted under the explicit fixture policy on
   DevNet/TestNet (evidence-only);
2. emergency council fixture execution accepted only under the explicit
   emergency fixture policy, and explicitly separate;
3. production governance execution remains unavailable/fail-closed;
4. on-chain governance execution remains unavailable/fail-closed;
5. MainNet governance execution remains unavailable/fail-closed;
6. governance input/decision/transcript/policy digests are deterministic and
   domain-bound;
7. governance execution authorizes a lifecycle action only when the action,
   candidate digest, and sequence all match;
8. validator-set rotation remains unsupported;
9. rejected governance-execution cases produce no mutation;
10. MainNet peer-driven apply remains refused even with fixture governance
    approval;
11. no real governance execution engine, on-chain proof verifier, KMS/HSM
    backend, RemoteSigner backend, or validator-set rotation is claimed.

## Real-binary surface

* `target/release/qbind-node --help` exposes no governance-execution surface and
  advertises no production / MainNet governance enablement and no on-chain
  governance proof verifier;
* default behaviour (no CLI, no env) does not expose or enable governance
  execution;
* every existing Run 070 / 130–211 surface
  (`--print-genesis-hash --env devnet|testnet|mainnet`, the Run 193 custody
  selector, the Run 198 RemoteSigner selector, the Run 209 custody-attestation
  selector, the governance on-chain fixture flag) stays governance-execution
  silent and preserves the MainNet peer-driven apply refusal even on MainNet.

## Reproduce

```bash
bash scripts/devnet/run_212_governance_execution_policy_release_binary.sh
```

The harness is idempotent: it wipes and regenerates everything under this
directory except `README.md`, `summary.txt`, and `.gitignore`.

## Honest limitations

* No real governance execution engine is implemented. Production / on-chain /
  MainNet governance execution always returns the typed unavailable outcome,
  regardless of inputs.
* No real on-chain governance proof verifier is implemented.
* Fixture governance execution remains DevNet/TestNet evidence-only and is
  refused on a MainNet trust domain.
* Emergency council fixture is explicit and non-production.
* The Run 211 boundary is consumed at the library level; the release binary
  does not yet wire a governance execution evaluator into a long-running node
  runtime — no production governance is enabled.
* No real KMS / HSM backend, no real RemoteSigner backend, and no production
  signing-key custody; KMS/HSM/RemoteSigner/custody-attestation remain
  boundary-only and unchanged. There is no schema / wire / metric drift.
* Validator-set rotation remains unsupported.
* MainNet peer-driven apply remains refused (Run 147 FATAL invariant) even with
  a fixture governance approval.
* Full C4 remains open. C5 remains open.
