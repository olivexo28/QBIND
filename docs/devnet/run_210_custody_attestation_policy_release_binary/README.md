## What this run proves

1. default behavior (no CLI, no env) resolves to
   `CustodyAttestationPolicy::Disabled`;
2. the hidden CLI selector resolves each canonical tag;
3. the hidden env selector resolves each canonical tag;
4. CLI-over-env precedence is deterministic;
5. invalid CLI/env selector values fail closed with typed parse errors;
6. the resolved policy reaches all seven production preflight contexts;
7. fixture custody attestation is accepted on DevNet/TestNet only where the
   resolved policy allows;
8. production / cloud-KMS / PKCS#11 / HSM / RemoteSigner attestation material
   reaches the Run 205 verifier and fails closed as unavailable regardless of
   the selected policy;
9. MainNet peer-driven apply remains refused even with
   `mainnet-production-attestation-required` and fixture attestation;
10. no real KMS / HSM attestation verifier / RemoteSigner backend / governance
    execution / real on-chain proof verifier / validator-set rotation is
    claimed.

## Honest limitations

* No real cloud KMS / PKCS#11 / HSM vendor / RemoteSigner attestation verifier
  is implemented. Carried production-class attestation material always routes
  into the Run 205 verifier and returns the typed unavailable outcome,
  regardless of the selected policy.
* Fixture custody attestation remains DevNet/TestNet evidence-only and is
  refused on a MainNet trust domain.
* The Run 209 CLI flag is consumed by the selector surface at the library
  level; the release binary parses it (hidden from `--help`) but does not yet
  wire its resolved policy into a long-running node runtime â€” arming it
  enables no production custody attestation.
* The hidden CLI/env selector is additive and disabled by default; an invalid
  selector value fails closed with a typed parse error. There is no schema /
  wire / metric drift.
* No real RemoteSigner backend / networked signer daemon is implemented;
  RemoteSigner and KMS/HSM remain backend-boundary only.
* governance execution remains unimplemented; real on-chain proof
  verification remains unimplemented; validator-set rotation remains open.
* MainNet peer-driven apply remains refused (Run 147 FATAL invariant) even
  with `mainnet-production-attestation-required` and fixture attestation
  material.
* Full C4 remains open. C5 remains open.
