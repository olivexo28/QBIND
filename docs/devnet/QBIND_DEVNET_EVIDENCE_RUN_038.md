### 9.1 Honest observability gap (recorded honestly, not silently fixed)

The `qbind_p2p_pqc_*` family of metrics declared in
`crates/qbind-node/src/metrics.rs::P2pMetrics::format_metrics` (lines
6230-6275) is **not** rendered by `NodeMetrics::format_metrics`
(grep: `self.p2p.format_metrics` is unreferenced in
`metrics.rs`), so the live `/metrics` HTTP endpoint served by
`metrics_http::format_metrics_output` (which calls
`metrics.format_metrics()` / `metrics.format_metrics_with_crypto(..)`)
**does not include**:

```
qbind_p2p_pqc_root_mode
qbind_p2p_pqc_roots_configured
qbind_p2p_pqc_cert_verify_accepted_total
qbind_p2p_pqc_cert_verify_rejected_total
qbind_p2p_pqc_cert_rejected_unknown_root_total
qbind_p2p_pqc_cert_rejected_wrong_suite_total
qbind_p2p_pqc_cert_rejected_bad_signature_total

qbind_p2p_pqc_cert_rejected_malformed_total

```

These metrics are exposed via the `P2pMetrics::format_metrics()` API
surface (Run 037 R037.I confirms the test-level export) but the live
binary `/metrics` endpoint does not currently include the
`P2pMetrics` family. **Run 038 records this as an honest evidence
inconsistency vs the Run 037 documentation** (which said the

`docs/whitepaper/contradiction.md` as a follow-up; **Run 038 does
not modify source code to fix it** because (a) no run-level decision

of `client handle_server_accept failed` / `server handle_client_init

discipline says "Do not modify source code unless the evidence run

## 12. B14 absent-leader fault description

  numeric PID (`11635`) at `2026-05-10T14:46:23Z`. V1A's process is
  the only one taken down; V0/V2A/V3A continue.

  `v % 4`. Therefore V1A is the round-robin leader for views

  next V1A-led view in the rotation is `105`, then `109`, `113`,
  `117`. Each of those views must time out and be advanced through a

- **Fault scope**: V0, V2A, V3A remain alive (≥ `2f+1 = 3` of 4
  validators). No operator intervention beyond the planned single
  `SIGINT`.
