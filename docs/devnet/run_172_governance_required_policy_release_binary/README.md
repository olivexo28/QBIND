production-surface evidence that real `target/release/qbind-node`
enforces `GovernanceProofPolicy::RequiredForLifecycleSensitive` via the
hidden Run 171 selector
(`--p2p-trust-bundle-governance-proof-required` /
`QBIND_P2P_TRUST_BUNDLE_GOVERNANCE_PROOF_REQUIRED=1`) on the production
v2 marker-decision surfaces.

## What this run proves on the real release binary

Real `target/release/qbind-node` was exercised with release-built
fixtures minted by the new `run_172_governance_required_policy_release_binary_helper`
example. The harness covers:

* **A1, A2** — default `NotRequired` compatibility (no selector, no
  proof) on validation-only `--p2p-trust-bundle-reload-check` and
  mutating `--p2p-trust-bundle-reload-apply-path`.
* **A3, A5** — Required policy via CLI / env on validation-only
  `--p2p-trust-bundle-reload-check` accepts a valid proof-carrying
  GenesisBound Rotate sidecar (no marker write, no sequence write).
* **A4, A6** — Required policy via CLI / env on mutating
  `--p2p-trust-bundle-reload-apply-path` accepts a valid
  proof-carrying GenesisBound Rotate sidecar; sequence commits and v2
  marker persists strictly after the sequence commit.
* **R2, R4** — Required policy + no-proof Rotate sidecar refused on
  the mutating preflight surface with
  `GovernanceAuthorityRequiredButMissing`; no Run 070 apply, no live
  trust mutation, no sequence commit, no marker persist.
* **R5, R9, R10, R11, R12, R13, R14, R18** — Required policy +
  malformed / wrong-root / wrong-action / wrong-digest /
  wrong-sequence / invalid-signature / unsupported-suite /
  OnChainGovernance proof-carrying sidecars all refused on the
  mutating preflight surface; no Run 070 apply log on any.
* **R23** — MainNet peer-driven apply refusal (Run 147 FATAL invariant)
  is preserved even with `--p2p-trust-bundle-governance-proof-required`
  and a valid proof-carrying Rotate sidecar.
* **R24** — selector cannot be implicitly enabled by unrelated flags
  (no-proof Rotate sidecar without selector accepted under
  NotRequired default; absence of `GovernanceAuthorityRequiredButMissing`).
* **A9, R21** — deferred to Run 168 release-built helper (in-process
  governance-proof matrix through production loader/gate symbols)
  and Run 161/165 source tests, because those scenarios cannot be
  minted as bit-for-bit static fixtures consumable by the binary.

The harness additionally:

* runs the Run 168 release-built helper end-to-end against the
  production loader/gate symbols;
* re-runs `cargo test --release -p qbind-node` for
  `run_171_governance_required_policy_selector_tests`,
  `run_169_governance_proof_loader_surface_integration_tests`,
  `run_167_governance_proof_carrier_tests`,
  `run_165_governance_marker_integration_tests`, and
  `--lib pqc_authority`;
* captures source-reachability greps for `governance_proof_policy_from_cli_or_env`,
  `governance_proof_required_env_selector_enabled`,
  `GovernanceProofPolicy::RequiredForLifecycleSensitive`,
  `preflight_v2_marker_decision_with_governance_proof_load`,
  `load_versioned_ratification_with_governance_proof_from_path`, and
  `GovernanceProofContext::Available`;
* enforces denylist greps over all logs (no MainNet apply, no
  autonomous apply, no apply on receipt, no peer-majority authority,
  no fallback to `--p2p-trusted-root`, no DummySig/DummyKem/DummyAead,
  no marker-before-sequence).

## Honest limitation (preserved from Run 170/171)

`preflight_run_132_validation_only_v2_marker_check` (the validation-only
reload-check / peer-candidate-check entry point in `main.rs`) does NOT
call `governance_proof_policy_from_cli_or_env`. The Required policy is
enforced on the mutating preflight surface
(`preflight_run_134_v2_marker_decision`,
`preflight_run_136_v2_marker_decision_for_startup`, the SIGHUP
live-reload preflight, and the peer-driven `ProductionV2MarkerCoordinator`
preflight). Validation-only surfaces parse the proof sibling through
the production loader (`load_versioned_ratification_with_governance_proof_from_path`)
but accept regardless of policy; the rejection branch on validation-only
is observed at symbol level via Run 168 helper + Run 169/171 tests.
